# Практика — урок 21

## Цель

Реализовать CPU-bound анализ набора byte buffers последовательно и через data parallelism, сохранив одинаковый результат и исключив общее изменяемое состояние.

## Baseline

Для каждого buffer вычислите deterministic checksum/statistics с достаточно заметной CPU-work. Последовательная версия возвращает массив результатов в порядке входа.

Зафиксируйте:

- размер/число inputs;
- expected aggregate;
- elapsed Release runs;
- allocations, если измеряете;
- machine logical processor count.

## Race experiment

Отдельно создайте небезопасный `count++` в Parallel.For. Запустите stress несколько раз и сохраните наблюдения. Затем исправьте через `Interlocked.Increment` только для независимого счётчика.

Объясните, почему Interlocked не решает произвольный многополевой invariant.

## Parallel implementation

Используйте `Parallel.For` с отдельным result slot по index. `ProcessBuffer` не изменяет static/shared state.

Поддержите:

- CancellationToken;
- MaxDegreeOfParallelism;
- корректный outcome при exception;
- порядок результатов, совпадающий с input.

## Эксперименты

Сравните degree 1, 2, число cores и заведомо избыточное значение на маленьких/средних/крупных inputs. Объясните overhead и saturation.

Создайте PLINQ-вариант с/без `AsOrdered`, сравните result contract и цену.

## Ограничения

- no shared List.Add из workers;
- no Thread.Sleep для coordination;
- no Task.Run вокруг каждой tiny operation;
- no unbounded tasks;
- одинаковые inputs/results для сравнения;
- Debug measurement не выдаётся за benchmark.

## Критерии приёмки

- baseline и parallel result идентичны;
- race experiment воспроизводит/объясняет lost update;
- cancellation наблюдается;
- failures не теряются;
- degree experiments задокументированы;
- speedup не заявляется без данных;
- README классифицирует CPU-bound и объясняет Task ≠ Thread.

## Дополнительное задание

Используйте partition-local aggregation и merge вместо отдельного result slot. Сравните contention и correctness с atomic global accumulator.
