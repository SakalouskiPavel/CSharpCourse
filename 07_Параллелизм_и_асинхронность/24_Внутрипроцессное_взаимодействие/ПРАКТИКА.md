# Практика — урок 24

## Цель

Создать bounded in-process pipeline обработки документов с несколькими producers/workers, backpressure, graceful completion и наблюдаемыми ошибками.

## Pipeline

Stages:

1. producers создают immutable `DocumentJob`;
2. bounded Channel принимает jobs;
3. N workers валидируют/обрабатывают;
4. results поступают в отдельный result Channel либо collector;
5. coordinator завершает input после всех producers;
6. host ждёт drain и завершение workers.

## Policies

До кода зафиксируйте:

- capacity и расчёт memory;
- `FullMode`;
- max workers;
- нужен ли input/completion order;
- item error vs fatal pipeline error;
- cancellation vs drain on shutdown;
- ownership message buffers;
- retries/dropped metrics.

## Backpressure test

Capacity=1, consumer заблокирован TaskCompletionSource. Первый Write проходит, второй обязан оставаться incomplete до чтения. Никаких Delay/Sleep.

## Completion tests

- normal: все buffered jobs обработаны;
- writer failure: readers получают cause;
- cancellation: workers завершаются по policy;
- один producer завершился раньше: channel остаётся доступным другим;
- early consumer exit освобождает owned resource.

## Ordering

Запустите минимум три workers с управляемым completion order. Покажите, что processing completion отличается от enqueue order. Если требуется order per CustomerId, partition jobs по стабильному key.

## Observability

Соберите metrics: accepted, completed, failed, dropped, active/max active, producer wait, processing time, queue age, shutdown duration.

## Ограничения

- no unbounded channel без доказанного bound;
- no polling ConcurrentQueue;
- no fire-and-forget workers;
- no mutable messages после Write;
- no hidden drop;
- no first-producer completion;
- all tasks observed.

## Критерии приёмки

- bounded behavior доказан deterministic test;
- completion ownership корректен;
- max concurrency не превышен;
- all errors/cancellation observable;
- graceful drain работает;
- order contract проверен;
- README сравнивает direct call, event, Channel и broker.

## Дополнительное задание

Реализуйте тот же pipeline на TPL Dataflow либо BlockingCollection и сравните async behavior, backpressure, completion и сложность.
