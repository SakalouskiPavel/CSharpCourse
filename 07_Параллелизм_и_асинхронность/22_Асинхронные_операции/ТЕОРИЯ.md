# Урок 22. Асинхронные операции

> Асинхронность освобождает поток во время ожидания и улучшает масштабируемость I/O-bound систем. Она не делает отдельную операцию автоматически быстрее и требует сквозного контракта отмены, timeout и ошибок.

## Результаты обучения

- объяснять runtime-модель темы;
- выбирать механизм по требуемой гарантии;
- сохранять ошибки, отмену и завершение;
- проверять корректность и производительность экспериментом.

## Карта темы

### 1. Task-based pattern

Async-метод возвращает Task/Task<T>, обычно имеет суффикс Async и принимает CancellationToken. Операция начинается при вызове, если API явно не говорит об обратном.

### 2. I/O и CPU

Настоящий async I/O ждёт completion ОС без worker. Task.Run переносит CPU-bound вычисление в pool, но не уменьшает работу и не нужен поверх уже асинхронного I/O.

### 3. Композиция

WhenAll ожидает все задачи, WhenAny — первую завершившуюся. Await внутри обычного цикла сериализует операции; одновременный запуск требует лимита.

### 4. Cancellation

Token — совместный сигнал, а не убийство потока. Он передаётся вниз, периодически проверяется и сохраняет отличимость отмены от ошибки.

### 5. Timeout

Timeout — политика ожидания вызывающего. Остановка ожидания не гарантирует остановку underlying operation, если ей не передан отменяемый token.

### 6. Throttling

SemaphoreSlim, Channel и Parallel.ForEachAsync ограничивают число активных операций в соответствии с лимитом сервиса, соединений и памяти.

### 7. ValueTask

Может уменьшить аллокации при частом синхронном результате, но сложнее в использовании и обычно ожидается один раз. Нужны измерения.

### 8. Async streams

IAsyncEnumerable выдаёт элементы по мере готовности. `await foreach` и cancellation позволяют streaming без материализации полного набора.

### 9. Ошибки

Ошибки хранятся в Task и проявляются при await. WhenAll должен быть ожидаем; fire-and-forget требует владельца и наблюдения результата.

## Минимальный пример

```csharp
await Parallel.ForEachAsync(urls,
    new ParallelOptions { MaxDegreeOfParallelism = 8, CancellationToken = ct },
    async (url, token) => await DownloadAsync(url, token));
```

Для конкурентного кода нарисуйте участников, разделяемые данные, точки ожидания и владельца завершения. Потокобезопасность отдельного вызова ещё не делает атомарной последовательность вызовов.

## Типичные ошибки

- путать Task и Thread;
- блокировать асинхронную работу через Wait или Result;
- запускать бесконтрольное число операций;
- терять исключение, cancellation или сигнал завершения;
- проверять гонки только единичным успешным запуском.

## Итог

Сначала докажите корректность простейшей версии, затем измеряйте и добавляйте параллелизм. Планировщик не обязан выбирать удобный interleaving, поэтому решение должно быть корректным при любом допустимом порядке.
