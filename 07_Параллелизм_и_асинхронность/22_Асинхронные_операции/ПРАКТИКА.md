# Практика — Урок 22. Асинхронные операции

## Задача

Создать ограниченный асинхронный загрузчик ресурсов с прогрессом, отменой, timeout и потоковой выдачей результатов.

## Обязательные требования

- использовать реальный async API без Task.Run для I/O;
- ограничить одновременные загрузки;
- различать cancellation, timeout и ошибку;
- не блокировать Tasks синхронно;
- покрыть координацию детерминированными тестами;

## Порядок работы

1. Запишите инварианты, участников и точки отмены.
2. Сделайте контрольный простой вариант.
3. Добавляйте concurrency по одному шагу.
4. Координируйте тесты сигналами, а не случайными задержками.
5. Проверьте Release-сборку и повторный стресс-запуск.

## Критерии готовности

- инварианты сохраняются при многократном выполнении;
- все фоновые операции имеют владельца;
- ошибки и отмена наблюдаются вызывающим кодом;
- shutdown завершается за ограниченное время;
- лимиты ресурсов заданы явно.

## Дополнительный уровень

- добавьте метрики времени ожидания и активной работы;
- опишите один альтернативный механизм и причину отказа от него.
