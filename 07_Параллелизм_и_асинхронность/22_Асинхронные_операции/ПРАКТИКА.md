# Практика — урок 22

## Цель

Создать bounded async loader локальных/HTTP-ресурсов с progress, cancellation, timeout и потоковой выдачей результатов.

## Контракт

```csharp
Task<ResourceResult> LoadAsync(
    ResourceRequest request,
    CancellationToken cancellationToken);
```

Result различает success, not found/validation outcome, timeout, cancellation и unexpected failure согласно документированному contract. Не превращайте все exceptions в null.

Используйте настоящий async stream/file/HTTP API. Task.Run вокруг него запрещён.

## Координация

`LoadManyAsync`:

- принимает sequence requests;
- ограничивает active operations;
- сохраняет input order в итоговом массиве **или** явно выдаёт completion order;
- не создаёт бесконтрольно tasks на весь огромный source;
- передаёт token каждой operation;
- применяет per-item timeout внутри общего cancellation budget.

Реализуйте вариант через Parallel.ForEachAsync, SemaphoreSlim с bounded input либо workers. Объясните выбор.

## Async stream

Создайте отдельный:

```csharp
IAsyncEnumerable<ResourceResult> LoadAsCompletedAsync(...)
```

Он выдаёт результаты по мере завершения. Документируйте, что происходит при ошибке/отмене после уже выданных элементов и как освобождаются resources при раннем break consumer-а.

## Progress

Сообщайте immutable progress: completed, total (если известен), active, bytes. Progress не должен менять core result и не должен публиковаться на каждый byte.

## Обязательные сценарии

- одна успешная operation;
- несколько независимых operations реально перекрываются;
- limit никогда не превышается (fake считает active/max);
- cancellation до старта и во время I/O;
- timeout отличается от user cancellation;
- одна failure не теряет outcomes остальных по выбранной policy;
- async stream consumer прекращает чтение рано;
- no unobserved background tasks.

## Детерминированные tests

Fake loader использует TaskCompletionSource для ручного завершения. Tests не используют sleeps для порядка. Каждый test имеет общий timeout.

## Критерии приёмки

- I/O не обёрнут в Task.Run;
- concurrency bounded и доказана счётчиком;
- cancellation/timeout проходят до dependency;
- ordering contract проверен;
- fire-and-forget отсутствует;
- async stream cleanup проверен;
- README сравнивает sequential, WhenAll unbounded и bounded design;
- solution собирается.
