# Урок 02. Данные, операторы и методы — от первой переменной до перегрузки

> Программа получает данные, преобразует их и возвращает результат. В этой главе мы сначала разберём типы, переменные, условия и циклы, которых нет отдельным пунктом в исходной программе курса, а затем научимся собирать действия в методы.

## Результаты обучения

После урока вы сможете:

- различать значение, тип, переменную и литерал;
- работать с числами, текстом и логическими значениями;
- использовать арифметические, сравнительные и логические операторы;
- управлять выполнением через `if`, `switch`, `for` и `while`;
- объявлять и вызывать методы;
- понимать параметры, аргументы и возвращаемое значение;
- объяснять передачу по значению, `ref`, `out` и `in`;
- проектировать понятные перегрузки и методы расширения;
- распознавать ситуации, где оператор перегружать не следует.

## 1. Значение, тип, переменная и литерал

Рассмотрим строку:

```csharp
int age = 25;
```

Здесь четыре разных понятия:

- `25` — **значение**, конкретные данные;
- `int` — **тип**, описание допустимых значений и операций;
- `age` — **переменная**, имя, с которым программа связывает значение;
- запись `25` прямо в коде — **литерал**.

Тип нужен компилятору, чтобы запретить бессмысленные операции и выбрать правильное представление данных. Возраст можно складывать с числом, но нельзя без правила сложить возраст и текст.

```csharp
int age = 25;
age = age + 1;
Console.WriteLine(age); // 26
```

`=` — оператор присваивания. Сначала вычисляется выражение справа `age + 1`, затем результат сохраняется в переменной слева. Это не математическое равенство.

## 2. Основные типы

### Целые числа

```csharp
int quantity = 12;
long worldPopulation = 8_000_000_000;
```

`int` — распространённый тип целых чисел. `long` имеет больший диапазон. Символ `_` внутри числового литерала улучшает чтение и не влияет на значение.

### Дробные числа

```csharp
double temperature = 36.6;
decimal price = 19.95m;
```

`double` хранит число с плавающей точкой и подходит для измерений и вычислений, допускающих погрешность. `decimal` имеет десятичную модель и обычно предпочтителен для денежных величин. Суффикс `m` сообщает компилятору, что литерал имеет тип `decimal`.

```csharp
Console.WriteLine(0.1 + 0.2); // double может показать приближение
```

Компьютер хранит многие дроби приближённо. Это не случайная ошибка: двоичное представление не может точно выразить некоторые десятичные дроби.

### Логическое значение

```csharp
bool isAdult = age >= 18;
```

`bool` имеет только два значения: `true` и `false`. Такие значения используются в условиях.

### Текст и символ

```csharp
string city = "Минск";
char firstLetter = 'М';
```

`string` хранит последовательность текста в двойных кавычках, `char` — одну единицу UTF-16 в одинарных. Подробности Unicode будут в уроке 07.

### `var` не означает отсутствие типа

```csharp
var score = 100;       // компилятор выводит int
var message = "Готово"; // компилятор выводит string
```

Тип определяется во время компиляции и больше не меняется. `var` удобно, когда тип очевиден справа; оно не превращает C# в язык с произвольными типами.

## 3. Преобразование и пользовательский ввод

`Console.ReadLine()` возвращает текст, даже если пользователь ввёл цифры. Чтобы получить число, текст нужно разобрать.

```csharp
Console.Write("Введите возраст: ");
string? input = Console.ReadLine();

if (int.TryParse(input, out int age))
{
    Console.WriteLine($"Через год вам будет {age + 1}");
}
else
{
    Console.WriteLine("Возраст должен быть целым числом");
}
```

`TryParse` пытается разобрать строку. Он возвращает `true` при успехе, а разобранное число помещает в `age` через параметр `out`. Неверный пользовательский ввод здесь является обычным ожидаемым исходом, поэтому исключение не требуется.

## 4. Операторы

Оператор — знак или комбинация знаков, задающая операцию.

### Арифметика

| Оператор | Смысл | Пример |
|---|---|---|
| `+` | сложение | `2 + 3` → `5` |
| `-` | вычитание | `7 - 2` → `5` |
| `*` | умножение | `4 * 3` → `12` |
| `/` | деление | `7 / 2` → `3` для `int` |
| `%` | остаток | `7 % 2` → `1` |

Целочисленное деление отбрасывает дробную часть:

```csharp
int wrongAverage = 7 / 2;       // 3
double average = 7.0 / 2.0;     // 3.5
```

### Сравнение

`==` и `!=` проверяют равенство/неравенство; `<`, `>`, `<=`, `>=` сравнивают порядок. Результат всегда `bool`.

```csharp
bool canEnter = age >= 18;
```

Не путайте `=` (сохранить значение) и `==` (сравнить значения).

### Логические операторы

```csharp
bool hasTicket = true;
bool isDoorOpen = true;
bool canEnter = hasTicket && isDoorOpen;
bool needsHelp = !canEnter;
```

- `&&` — оба условия истинны;
- `||` — хотя бы одно истинно;
- `!` — отрицание.

`&&` и `||` вычисляются сокращённо: правая часть не запускается, если ответ уже известен. Это позволяет безопасно проверить значение перед обращением к нему:

```csharp
if (name is not null && name.Length > 0)
{
    Console.WriteLine(name);
}
```

## 5. Ветвление: `if` и `switch`

```csharp
if (age < 0)
{
    Console.WriteLine("Возраст не может быть отрицательным");
}
else if (age < 18)
{
    Console.WriteLine("Несовершеннолетний пользователь");
}
else
{
    Console.WriteLine("Совершеннолетний пользователь");
}
```

Условия проверяются сверху вниз. Выполняется первая подходящая ветка, остальные пропускаются. Фигурные скобки задают блок инструкций.

`switch` удобен, когда один результат сопоставляется с вариантами:

```csharp
string roleName = role switch
{
    "admin" => "Администратор",
    "editor" => "Редактор",
    "reader" => "Читатель",
    _ => "Неизвестная роль"
};
```

`_` означает любой оставшийся вариант. Switch expression возвращает значение и заставляет явно подумать о неизвестном вводе.

## 6. Повторение: циклы

### `for`

```csharp
for (int number = 1; number <= 5; number++)
{
    Console.WriteLine(number);
}
```

В заголовке:

1. `int number = 1` выполняется один раз;
2. `number <= 5` проверяется перед каждой итерацией;
3. `number++` увеличивает число после тела.

### `while`

```csharp
string? command = Console.ReadLine();
while (command != "exit")
{
    Console.WriteLine($"Получена команда: {command}");
    command = Console.ReadLine();
}
```

`while` подходит, когда число повторений заранее неизвестно. Если условие никогда не станет ложным, получится бесконечный цикл.

## 7. Зачем нужны методы

Без методов программа быстро превращается в длинный список действий. **Метод** даёт фрагменту имя, входные данные и результат.

```csharp
decimal CalculateTotal(decimal price, int quantity)
{
    if (price < 0)
        throw new ArgumentOutOfRangeException(nameof(price));
    if (quantity < 0)
        throw new ArgumentOutOfRangeException(nameof(quantity));

    return price * quantity;
}

decimal total = CalculateTotal(19.95m, 3);
Console.WriteLine(total); // 59.85
```

Разбор объявления:

- первый `decimal` — тип возвращаемого результата;
- `CalculateTotal` — имя, описывающее действие;
- в скобках объявлены параметры `price` и `quantity`;
- `return` завершает метод и возвращает вычисленное значение.

При вызове `19.95m` и `3` называются аргументами. Параметр — место в объявлении, аргумент — конкретное значение при вызове.

Метод без результата объявляется с `void`:

```csharp
void PrintSeparator()
{
    Console.WriteLine("----------------");
}
```

## 8. Передача аргументов

По умолчанию C# передаёт **копию значения переменной**.

```csharp
void AddOne(int number)
{
    number++;
}

int count = 5;
AddOne(count);
Console.WriteLine(count); // 5
```

Метод изменил свою локальную копию. Для ссылочного типа копируется ссылка на объект: обе переменные могут указывать на один изменяемый объект, но сама переменная вызывающего кода по-прежнему не заменяется.

Специальные модификаторы:

- `ref` — метод читает и может заменить существующую переменную;
- `out` — метод обязан записать выходное значение до завершения;
- `in` — передача по ссылке только для чтения, полезна главным образом для крупных структур.

```csharp
bool TryDivide(decimal left, decimal right, out decimal result)
{
    if (right == 0)
    {
        result = 0;
        return false;
    }

    result = left / right;
    return true;
}
```

Не применяйте `ref` только ради демонстрации. Возвращаемое значение обычно читается проще.

## 9. Перегрузка методов

Несколько методов могут иметь одно имя, если различаются списком параметров:

```csharp
decimal CalculateTotal(decimal price, int quantity) => price * quantity;

decimal CalculateTotal(decimal price, int quantity, decimal discountPercent)
{
    decimal total = price * quantity;
    return total * (1 - discountPercent / 100);
}
```

Компилятор выбирает подходящую **перегрузку** по числу, типам и модификаторам параметров. Только возвращаемого типа недостаточно: вызывающий код может не использовать результат, и выбор стал бы неоднозначным.

Хорошие перегрузки выполняют одну смысловую операцию. Если варианты делают принципиально разные вещи, дайте им разные имена.

## 10. Методы расширения

Extension method позволяет вызвать внешний статический метод так, будто он является членом типа.

```csharp
public static class StringExtensions
{
    public static bool HasText(this string? value)
    {
        return !string.IsNullOrWhiteSpace(value);
    }
}

bool valid = name.HasText();
```

Класс и метод должны быть `static`, первый параметр помечается `this`. Тип `string` не изменился: компилятор преобразует вызов в `StringExtensions.HasText(name)`.

Расширение уместно, если операция естественна для типа и находится в понятном пространстве имён. Оно не получает доступ к закрытым полям и уступает настоящему instance method с той же сигнатурой.

## 11. Пользовательские операторы

Для собственного типа можно определить естественную арифметику:

```csharp
public readonly record struct Money(decimal Amount, string Currency)
{
    public static Money operator +(Money left, Money right)
    {
        if (left.Currency != right.Currency)
            throw new InvalidOperationException("Нельзя складывать разные валюты");

        return new Money(left.Amount + right.Amount, left.Currency);
    }
}
```

`operator +` должен вести себя так, как пользователь ожидает от сложения: не изменять незаметно операнды, быть предсказуемым и согласованным с предметной областью. Для конвертации валют нужен курс и момент времени, поэтому обычный именованный метод честнее простого `+`.

Если перегружаются `==` и `!=`, равенство следует согласовать с `Equals` и `GetHashCode`, иначе коллекции и сравнение будут противоречить друг другу.

## 12. Частые ошибки новичка

- `=` используется вместо `==` или наоборот;
- результат целочисленного деления неожиданно теряет дробную часть;
- деньги хранятся в `double` без понимания погрешности;
- пользовательский ввод преобразуется через `Parse` без обработки неверного текста;
- цикл не изменяет переменную условия и не заканчивается;
- метод имеет имя `DoWork`, хотя выполняет конкретный расчёт;
- `ref` применяется вместо обычного возвращаемого значения;
- перегрузки с одинаковым именем имеют разный смысл;
- extension methods превращаются в склад несвязанной логики;
- пользовательский оператор выглядит коротко, но скрывает I/O или дорогое вычисление.

## 13. Мини-проверки

1. Что выведет `Console.WriteLine(5 / 2)` и почему?
2. Чем параметр отличается от аргумента?
3. Изменится ли `count` после передачи обычного `int` в метод?
4. Почему нельзя выбрать перегрузку только по возвращаемому типу?
5. Изменяет ли extension method исходный класс?

<details>
<summary>Краткие ответы</summary>

1. `2`: оба операнда целые, поэтому выполняется целочисленное деление.
2. Параметр объявляется методом, аргумент передаётся при конкретном вызове.
3. Нет: метод получает копию значения, если не использован `ref`.
4. Вызов может игнорировать результат, поэтому компилятору нечем однозначно выбрать метод.
5. Нет. Это специальный синтаксис вызова доступного статического метода.

## Итоговая карта

```text
Тип задаёт допустимые значения и операции
  ↓
Переменная связывает имя со значением
  ↓ операторы образуют выражения
Условия выбирают ветку, циклы повторяют действия
  ↓
Метод даёт действию имя, параметры и результат
  ↓
Перегрузка предлагает варианты одного действия
  ↓
Extension method добавляет удобную внешнюю операцию
```

Перед следующим уроком вы должны уметь написать небольшую консольную программу с вводом числа, проверкой через `TryParse`, условием, циклом и двумя собственными методами.
