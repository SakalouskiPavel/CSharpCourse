# Урок 02. Операторы и методы: перегрузка и расширения

> Метод — основная единица поведения в C#, а операторы дают типам естественный синтаксис. Урок связывает сигнатуры, передачу аргументов, перегрузку, пользовательские операторы и extension methods с правилами проектирования понятного API.

## Результаты обучения

- проектировать сигнатуры методов и перегрузки
- объяснять передачу по значению, `ref`, `out` и `in`
- перегружать операторы без нарушения ожиданий
- писать и обнаруживать методы расширения

## Карта темы

### 1. Сигнатура метода

Для перегрузки значимы имя, число параметров, их типы и модификаторы передачи; возвращаемый тип сам по себе перегрузку не образует. Необязательные и именованные аргументы влияют на вызов, но требуют осторожности при развитии API.

### 2. Параметры и значения

Обычный параметр получает копию переменной: для ссылочного типа копируется ссылка. `ref` позволяет читать и менять существующую переменную, `out` требует присвоения в методе, `in` передаёт значение по ссылке только для чтения.

### 3. Перегрузка

Компилятор выбирает лучший применимый кандидат с учётом неявных преобразований. Набор перегрузок должен означать одну операцию; неоднозначность часто возникает рядом с `null`, generic-методами и необязательными параметрами.

### 4. Локальные функции и expression body

Локальная функция скрывает вспомогательный алгоритм внутри метода и может захватывать окружение. Выражение `=>` уместно для короткой логики, но не должно ухудшать чтение.

### 5. Пользовательские операторы

Операторы объявляются `public static`; хотя бы один операнд должен иметь содержащий тип. Для значимых объектов полезно согласовывать `==`, `!=`, `Equals` и `GetHashCode`.

### 6. Преобразования типов

`implicit` допустим только для очевидного и безопасного преобразования без потери данных. Потенциально ошибочное или дорогое преобразование должно быть `explicit` либо обычным именованным методом.

### 7. Методы расширения

Это статические методы статического класса, где первый параметр помечен `this`. Они не меняют тип и уступают настоящим экземплярным методам; особенно полезны для небольших операций над интерфейсами.

### 8. Аргументная валидация

Публичный метод защищает свои предусловия и выбрасывает подходящие исключения. Современные guard-методы вроде `ArgumentNullException.ThrowIfNull` сокращают шум.

## Минимальный пример

```csharp
public readonly record struct Money(decimal Amount, string Currency)
{
    public static Money operator +(Money left, Money right)
    {
        if (left.Currency != right.Currency)
            throw new InvalidOperationException("Currencies differ");
        return left with { Amount = left.Amount + right.Amount };
    }
}

public static class MoneyExtensions
{
    public static bool IsPositive(this Money money) => money.Amount > 0;
}
```

Разберите пример построчно: какие гарантии выражены типами, где проверяются предусловия и что увидит вызывающий код. Затем измените один инвариант и проследите, какая часть публичного контракта должна поменяться.

## Типичные ошибки

- использовать `ref` вместо возвращаемого значения без необходимости;
- создавать перегрузки с неожиданно разным смыслом;
- делать неявными преобразования, которые могут потерять данные;
- размещать бизнес-логику в случайных extension-классах;

## Итог

Тема считается усвоенной, когда вы можете не только применить механизм, но и объяснить его цену, область корректности и более простой альтернативный вариант. Проверяйте решения сборкой, тестами и небольшими экспериментами, а не только чтением.
