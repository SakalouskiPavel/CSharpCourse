# Практика — урок 02

## Цель

Создать консольный калькулятор заказа. Задание последовательно проверяет типы, ввод, операторы, условия, циклы и методы. Оно не требует классов, records, коллекций или test framework: эти темы изучаются позднее.

## Сценарий

Программа несколько раз запрашивает цену и количество товара, рассчитывает стоимость позиции и накапливает итог. В конце применяется скидка и печатается чек.

## Этап 1. Одна позиция

Запросите:

- название товара — `string?`;
- цену — `decimal` через `decimal.TryParse`;
- количество — `int` через `int.TryParse`.

Цена должна быть больше нуля, количество — положительным целым. При ошибке выведите понятное сообщение и завершите текущую попытку без необработанного исключения.

Создайте метод:

```csharp
decimal CalculateLineTotal(decimal price, int quantity)
```

Он проверяет аргументы и возвращает произведение.

## Этап 2. Несколько позиций

Организуйте цикл. После каждой позиции задавайте вопрос, продолжать ли ввод. Хранить все позиции пока не нужно: достаточно переменной `decimal orderTotal`, к которой прибавляется сумма очередной позиции.

Цикл обязан завершаться по команде пользователя. Проверьте ввод одной, трёх и нуля корректных позиций.

## Этап 3. Скидка

Создайте две перегрузки с одним смыслом:

```csharp
decimal ApplyDiscount(decimal total, decimal percent)
decimal ApplyDiscount(decimal total, decimal percent, decimal maximumDiscount)
```

Правила:

- итог не может быть отрицательным;
- процент находится от 0 до 100;
- первая перегрузка применяет процент без дополнительного ограничения;
- вторая ограничивает абсолютную сумму скидки параметром `maximumDiscount`;
- методы возвращают новую сумму и не печатают текст.

## Этап 4. Логические условия

Скидка доступна, если сумма не меньше заданного порога **и** пользователь ввёл промокод. Выразите правило через `&&`. Пустой промокод проверяйте безопасно.

Для диапазона итоговой суммы выведите категорию через `switch` expression: маленький, обычный или крупный заказ. Границы категорий запишите в README.

## Этап 5. Метод расширения

Добавьте ровно один extension method для текста:

```csharp
bool HasText(this string? value)
```

Он возвращает `false` для `null`, пустой строки и строки из пробелов. Используйте его при проверке промокода. В README покажите эквивалентный обычный статический вызов и объясните, что тип `string` не изменился.

До запуска заполните ожидаемые результаты, после запуска — фактические:

## Обязательные ограничения

- не использовать `double` для денежных значений;
- не использовать `Parse` для непроверенного пользовательского ввода;
- не дублировать формулу позиции в `Program.cs`;
- не применять `ref` или `out`, кроме `TryParse`;
- не создавать классы предметной области до урока 03;
- каждая перегрузка `ApplyDiscount` выполняет одну и ту же смысловую операцию;
- программа собирается без ошибок.

## Критерии приёмки

- `dotnet build` завершается успешно;
- корректный ввод даёт рассчитанный чек;
- неверный ввод не завершает программу необработанным исключением;
- цикл завершается по явной команде;
- присутствуют и используются требуемые методы и перегрузки;
- таблица содержит ожидаемые и фактические результаты;
- ученик может объяснить тип каждой переменной и каждое условие.
