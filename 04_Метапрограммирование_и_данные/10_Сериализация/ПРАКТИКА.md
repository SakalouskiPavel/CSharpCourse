# Практика — урок 10

## Цель

Создать версионируемое JSON-хранилище каталога книг, отделив transport DTO от domain model и сохранив совместимость со старым форматом.

## Контракты

Создайте:

- текущий `BookDtoV2`: number, title, author, decimal price, tags, optional publisher;
- старый `BookDtoV1`, где нет publisher, а tags могут отсутствовать;
- domain `Book`, который защищает непустые значения, цену и уникальность тегов.

Не ставьте JSON-атрибуты на domain `Book`. Реализуйте явные `ToDto` и `ToDomain`/factory mapping.

## Единые options

Один экземпляр options задаёт:

- camelCase naming;
- читаемое форматирование для учебного файла;
- ограничение глубины;
- enum representation, если enum используется;
- custom converter для `InventoryNumber` или другого value object.

README фиксирует контракт примерами JSON.

## Операции хранилища

- async save с CancellationToken;
- async load;
- запись во временный файл и безопасная замена основной версии;
- понятное различение: файл отсутствует, JSON повреждён, DTO невалиден, I/O failure;
- отсутствие частично созданного domain catalog при ошибке одного элемента.

## Совместимость

Поддержите чтение V1 и преобразование в текущую модель:

- отсутствующие tags становятся пустым набором;
- publisher остаётся отсутствующим;
- неизвестные дополнительные properties не ломают reader;
- сохранение всегда выполняется в V2.

Добавьте поле version либо обоснуйте другой способ определения версии. Не угадывайте версию по случайному отсутствию данных, если это неоднозначно.

## Обязательные сценарии

1. round trip V2 сохраняет значимые данные;
2. V1 успешно мигрирует;
3. неизвестное поле игнорируется;
4. отрицательная цена отклоняется после syntactic deserialize;
5. повреждённый JSON даёт отдельную ошибку;
6. JSON `null` не превращается в пустой корректный каталог молча;
7. неизвестная версия отклоняется;
8. отмена не оставляет основной файл частично записанным;
9. converter читает и пишет одинаковый value object.

## Ограничения

- domain objects не сериализуются напрямую;
- ошибки не проглатываются через catch-all;
- options не создаются заново с разными настройками;
- тип не выбирается через произвольное CLR-name из JSON;
- secret/internal fields отсутствуют в DTO;
- вход ограничен по размеру до десериализации.

## Критерии приёмки

- JSON-контракт документирован независимо от domain implementation;
- V1/V2 проверены сохранёнными sample payloads;
- mapping защищает инварианты;
- типы ошибок различимы вызывающим кодом;
- сохранение устойчиво к отмене/сбою до replace;
- round-trip и negative scenarios воспроизводятся;
- README описывает additive и breaking изменения.

## Дополнительное задание

Добавьте source-generated JsonSerializerContext и сравните поддерживаемые сценарии с reflection mode. Затем реализуйте чтение большого массива через `DeserializeAsyncEnumerable<T>` и определите политику partial failure, не заменяя простой вариант без измеримой причины.
