# Практика — урок 08

## Цель

Создать безопасный registry консольных команд: найти реализации общего интерфейса в явно выбранной сборке, проверить их и затем выполнять через типизированный контракт.

## Контракт

```csharp
public interface ICommandHandler
{
    string Name { get; }
    string Description { get; }
    void Execute(string[] arguments);
}
```

Поместите контракт в отдельный project либо в библиотеку приложения. Реализуйте минимум три команды.

## Discovery

Registry получает конкретный `Assembly`, а не сканирует весь процесс. Найдите типы, которые:

- являются concrete classes;
- реализуют `ICommandHandler`;
- не являются open generic types;
- имеют публичный конструктор без параметров для базовой версии задания.

## Validation

До запуска любой команды соберите все ошибки конфигурации:

- отсутствующий конструктор;
- пустое имя;
- дубликат имени без учёта регистра;
- тип, который бросает исключение при создании;
- abstract handler;
- тип, не реализующий контракт.

Ошибки должны указывать тип и правило. Registry либо создаётся полностью корректным, либо сообщает совокупную диагностику.

## Execution

После discovery храните `IReadOnlyDictionary<string,ICommandHandler>` с ordinal-ignore-case comparer. Команду вызывайте через `ICommandHandler.Execute`, а не через MethodInfo.Invoke.

Добавьте `help`, автоматически выводящий имя и описание всех зарегистрированных команд.

## Reflection report

Отдельная диагностическая команда принимает разрешённый тип из registry и выводит:

- full type name;
- assembly name;
- implemented interfaces;
- публичные свойства;
- публичные методы, объявленные самим типом;
- параметры выбранного метода.

Используйте явные BindingFlags и объясните каждый.

## Ошибка handler-а

Создайте отдельный учебный вызов через `MethodInfo.Invoke`, handler которого выбрасывает исключение. Покажите `TargetInvocationException` и inner exception. Основной dispatcher при этом остаётся типизированным.

## Ограничения

- assembly задаётся приложением, а не пользовательским путём/именем типа;
- scanning выполняется один раз;
- private members не используются для рабочего контракта;
- `Type.GetType(userInput)` запрещён;
- все найденные metadata/descriptors кэшируются registry;
- сообщения не скрывают исходную причину ошибки.

## Критерии приёмки

- три команды обнаруживаются автоматически в выбранной assembly;
- все ошибочные типы получают понятную диагностику;
- дубликаты обнаруживаются до выполнения;
- dispatcher после старта не использует Reflection для вызова;
- diagnostic report корректно различает inherited/declared members;
- README описывает runtime-риски, trimming и альтернативу без Reflection;
- solution собирается, сценарии повторяются.

## Дополнительное задание

Создайте delegate из `MethodInfo` один раз и сравните форму вызова с `Invoke`. Затем опишите архитектуру collectible AssemblyLoadContext, не добавляя динамическую загрузку в базовое решение без необходимости.
