# Практика — Урок 27. Архитектура проекта

## Задача

Спроектировать и реализовать небольшой сервис заказов с явными границами domain, use cases и infrastructure.

## Обязательные требования

- зафиксировать quality attributes и контекстную диаграмму;
- не допустить зависимости domain от infrastructure;
- сделать один сквозной vertical slice;
- собрать зависимости только в composition root;
- добавить architecture test и ADR;

## Порядок работы

1. Опишите use case, границу доверия и качественные требования.
2. Зафиксируйте контракт до инфраструктурной реализации.
3. Реализуйте один сквозной сценарий и негативный сценарий.
4. Добавьте integration/architecture tests.
5. Документируйте запуск, данные, секреты и принятое решение.

## Критерии готовности

- границы и зависимости видны из структуры;
- security checks выполняются server-side;
- ошибки не раскрывают внутренние данные;
- тесты воспроизводят отказ инфраструктуры;
- решение имеет наблюдаемость и инструкцию запуска.

## Дополнительный уровень

- сравнить layered и vertical-slice раскладку на одном изменении;
- добавить transactional outbox как отдельное решение, если обосновано;
