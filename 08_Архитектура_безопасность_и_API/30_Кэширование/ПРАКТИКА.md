# Практика — Урок 30. Волшебство кэширования без волшебства

## Задача

Добавить безопасное кэширование чтения каталога с измеримой политикой свежести и защитой от stampede.

## Обязательные требования

- описать key, TTL, invalidation и допустимую stale-данность;
- не смешивать tenants/users;
- реализовать single-flight на ключ;
- не падать полностью при недоступности cache;
- добавить метрики и tests с управляемым clock;

## Порядок работы

1. Опишите use case, границу доверия и качественные требования.
2. Зафиксируйте контракт до инфраструктурной реализации.
3. Реализуйте один сквозной сценарий и негативный сценарий.
4. Добавьте integration/architecture tests.
5. Документируйте запуск, данные, секреты и принятое решение.

## Критерии готовности

- границы и зависимости видны из структуры;
- security checks выполняются server-side;
- ошибки не раскрывают внутренние данные;
- тесты воспроизводят отказ инфраструктуры;
- решение имеет наблюдаемость и инструкцию запуска.
