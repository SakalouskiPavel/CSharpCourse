# Практика — Урок 29. WCF, ASMX, Web API и REST

## Задача

Создать HTTP API каталога с корректной семантикой ресурсов, ошибок, concurrency и документацией.

## Обязательные требования

- использовать HTTP methods/status codes по смыслу;
- возвращать ProblemDetails без внутренних деталей;
- поддержать validation и cancellation;
- добавить ETag/If-Match или idempotency key;
- проверить OpenAPI и API integration tests;

## Порядок работы

1. Опишите use case, границу доверия и качественные требования.
2. Зафиксируйте контракт до инфраструктурной реализации.
3. Реализуйте один сквозной сценарий и негативный сценарий.
4. Добавьте integration/architecture tests.
5. Документируйте запуск, данные, секреты и принятое решение.

## Критерии готовности

- границы и зависимости видны из структуры;
- security checks выполняются server-side;
- ошибки не раскрывают внутренние данные;
- тесты воспроизводят отказ инфраструктуры;
- решение имеет наблюдаемость и инструкцию запуска.

## Дополнительный уровень

- создать небольшой SOAP-адаптер к одному use case либо описать migration anti-corruption layer;
- добавить consumer contract test;
