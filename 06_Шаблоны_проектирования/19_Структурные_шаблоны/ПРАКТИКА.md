# Практика — Урок 19. Структурные шаблоны проектирования

## Задача

Создать единый контракт хранилища, адаптер legacy API и цепочку инфраструктурных оболочек.

## Обязательные требования

- реализовать Adapter с переводом ошибок;
- добавить logging и metrics decorators;
- добавить cache proxy с invalidation policy;
- сохранить cancellation и исключения;
- проверить порядок оболочек тестами;

## Порядок работы

1. Зафиксируйте оси изменения и инварианты.
2. Реализуйте минимальный сквозной сценарий.
3. Добавляйте паттерны по одному, сохраняя тесты зелёными.
4. Опишите выбор и отвергнутую более простую альтернативу.
5. Выполните `dotnet build` и `dotnet test`.

## Критерии готовности

- каждый паттерн решает названную проблему;
- зависимости и владение объектами явны;
- ошибочные варианты проверены тестами;
- нет глобального изменяемого состояния и декоративных абстракций.

## Дополнительный уровень

- выделить Bridge между типом отчёта и renderer;
