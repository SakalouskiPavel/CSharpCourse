# Практика — Урок 18. Порождающие шаблоны проектирования

## Задача

Создать генератор документов с двумя согласованными семействами renderer/exporter и пошаговой сборкой документа.

## Обязательные требования

- реализовать Abstract Factory для двух семейств;
- использовать Builder для валидной структуры документа;
- не обращаться к DI container из рабочих классов;
- показать один простой named factory;
- покрыть несовместимые конфигурации тестами;

## Порядок работы

1. Зафиксируйте оси изменения и инварианты.
2. Реализуйте минимальный сквозной сценарий.
3. Добавляйте паттерны по одному, сохраняя тесты зелёными.
4. Опишите выбор и отвергнутую более простую альтернативу.
5. Выполните `dotnet build` и `dotnet test`.

## Критерии готовности

- каждый паттерн решает названную проблему;
- зависимости и владение объектами явны;
- ошибочные варианты проверены тестами;
- нет глобального изменяемого состояния и декоративных абстракций.

## Дополнительный уровень

- добавить Prototype для шаблонов с доказанным deep copy;
- описать, какие порождающие паттерны сознательно не применены;
