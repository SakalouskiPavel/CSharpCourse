# Практика — урок 01

## Цель

Самостоятельно пройти полный первый цикл: создать проект, изменить код, прочитать результат сборки, запустить программу, оформить инструкцию и сохранить работу в Git. Задание использует только конструкции, объяснённые в теории урока.

## Итоговый результат

Консольная программа спрашивает имя и учебную цель, после чего печатает персональную карточку:

```text
Как вас зовут? Анна
Какую цель вы ставите на курс? Научиться создавать приложения

=== Карточка ученика ===
Имя: Анна
Цель: Научиться создавать приложения
Платформа: .NET
========================
```

## Часть 1. Проверка среды

1. Откройте терминал в каталоге урока.
2. Выполните `dotnet --info`.
3. Сохраните в README проекта номер установленного SDK и операционную систему. Не копируйте весь технический вывод.
4. Выполните `git status` и своими словами запишите, что он показывает.

## Часть 2. Создание solution и проекта

```powershell
dotnet new sln -n Lesson01
dotnet new console -n StudentCard
dotnet sln Lesson01.sln add StudentCard/StudentCard.csproj
dotnet build Lesson01.sln
```

После каждой команды объясните, какой файл или каталог появился. Если команда завершилась ошибкой, сохраните полный текст до попытки исправления.

## Часть 3. Программа

Используйте `Console.Write`, `Console.ReadLine`, переменные типа `string?`, интерполяцию и `Console.WriteLine`. Не требуется применять условия, циклы, классы или автоматические тесты: они ещё не изучены.

Обязательные требования:

- запросить имя;
- запросить учебную цель;
- сохранить оба ответа в отдельных переменных;
- вывести карточку не менее чем из четырёх строк;
- использовать интерполяцию `$"...{value}..."`;
- не заменять введённые данные заранее написанным текстом.

## Часть 4. Намеренная ошибка

1. Удалите одну закрывающую кавычку.
2. Выполните `dotnet build`.
3. Найдите код ошибки, файл и строку.
4. Сохраните пример диагностики в README и объясните её одной фразой.
5. Исправьте код и снова получите `Build succeeded`.

Цель шага — научиться читать ошибку, а не избегать ошибок.

## Часть 5. README

В каталоге `StudentCard` создайте `README.md` со следующими разделами:

- назначение программы;
- требования к среде;
- команды сборки и запуска;
- пример диалога;
- критерии приёмки;
- Definition of Done для этой учебной работы;
- разобранная ошибка компиляции;
- короткая ретроспектива: что получилось, где возникла трудность, что попробовать в следующем уроке.

## Часть 6. Git

Сделайте минимум два логических коммита:

1. solution и запускаемый минимальный проект;
2. карточка ученика и документация.

Перед каждым коммитом выполните `git status`. Проверьте, что `bin` и `obj` не попали в staging area.

## Критерии приёмки

- `dotnet build Lesson01.sln` завершается успешно;
- `dotnet run --project StudentCard` позволяет ввести два значения и выводит их;
- README позволяет повторить запуск без устных подсказок;
- в Git есть минимум два осмысленных коммита;
- `bin` и `obj` не отслеживаются;
- ученик может объяснить каждую строку `Program.cs`.

## Дополнительное задание

Измените оформление карточки, используя только уже знакомые операции вывода. До запуска письменно предскажите точный результат, затем сравните его с фактическим.

Автоматические тесты и CI здесь намеренно не требуются: соответствующие инструменты ещё не введены теорией.
