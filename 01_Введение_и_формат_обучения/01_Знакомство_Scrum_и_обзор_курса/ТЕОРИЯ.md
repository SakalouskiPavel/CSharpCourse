# Урок 01. Первое знакомство с программированием, .NET, Git и учебным процессом

> Это стартовая глава для человека, который раньше не писал программы. Здесь нет требования «уже понимать компьютер». К концу урока вы создадите и запустите первую программу, узнаете назначение основных файлов и сохраните результат в Git.

## Что получится после урока

Вы сможете:

- объяснить разницу между программой, исходным кодом, C# и .NET;
- проверить установку .NET SDK и прочитать базовый вывод команды;
- отличить терминал, IDE, проект и solution;
- создать, собрать и запустить консольное приложение;
- найти точку входа и изменить вывод;
- понимать назначение Git, ветки и коммита;
- описать учебную задачу через цель и критерии готовности;
- не теряться при первой ошибке компиляции.

## 1. Что такое программа

Компьютер хорошо выполняет точные инструкции и плохо понимает подразумеваемый смысл. Человек поймёт просьбу «покажи приветствие», а машине нужно сообщить:

- где взять имя;
- какой текст вывести;
- куда его вывести;
- что делать, если имя не введено;
- когда работа закончена.

**Программа** — набор инструкций и данных для выполнения задачи. **Исходный код** — текст этих инструкций, доступный человеку. **Язык программирования** задаёт словарь и правила, по которым исходный код записывается.

C# произносится «си шарп». Это язык общего назначения: на нём создают серверные приложения, сайты, настольные программы, игры, инструменты автоматизации и многое другое.

### Аналогия и её граница

Код похож на кулинарный рецепт: действия идут в определённом порядке и используют ингредиенты-данные. Но компьютер не повар: он не догадается, что значит «немного» или «до готовности». Условия должны быть формализованы.

## 2. Где в этой картине .NET

C# — язык, а **.NET** — платформа, на которой программа строится и выполняется. У платформы несколько важных частей:

1. **SDK (Software Development Kit)** — инструменты разработчика. В него входят компилятор, шаблоны и команда `dotnet`.
2. **Runtime** — среда выполнения, которая запускает уже собранное приложение, управляет памятью и сообщает об ошибках выполнения.
3. **Base Class Library** — стандартная библиотека готовых типов для текста, файлов, сети, коллекций и других типовых задач.

Упрощённый путь программы выглядит так:

```text
Исходный код C#
      ↓ компилятор проверяет и переводит
Сборка .NET
      ↓ runtime загружает и выполняет
Результат работы программы
```

Компилятор не знает, чего хотел заказчик. Он может обнаружить нарушенные правила языка, но не поймёт, что скидка посчитана неправильно. Поэтому кроме сборки нужны примеры и тесты.

## 3. IDE и терминал — два способа управлять инструментами

**IDE** — программа-редактор для разработки. Она подсвечивает код, предлагает дополнения, показывает ошибки, запускает debugger. Visual Studio и Rider — IDE; VS Code с расширениями ближе к настраиваемому редактору.

**Терминал** — текстовое окно для команд. Команда:

```powershell
dotnet --info
```

означает: найти программу `dotnet`, запустить её и передать аргумент `--info`. Команда ничего не меняет: она печатает сведения об установленных SDK и runtime.

Не нужно выбирать «IDE или терминал». Они дополняют друг друга: IDE удобна для кода и отладки, CLI-команды воспроизводимы на другом компьютере и в CI.

### Мини-проверка

Какую часть вы проверяете командой `dotnet --info`: язык C#, SDK или файл проекта?

<details>
<summary>Ответ</summary>

Команда проверяет установленный инструмент .NET CLI и показывает SDK/runtime. Язык C# не является отдельной запускаемой программой, а проект ещё не создан.

## 4. Создаём первую программу

Откройте терминал в каталоге текущего урока и выполните:

```powershell
dotnet new console -n FirstProgram
```

Разберём команду:

- `dotnet` — запускаемый инструмент;
- `new` — создать по шаблону;
- `console` — шаблон консольного приложения;
- `-n FirstProgram` — имя проекта и нового каталога.

Перейдите в каталог:

```powershell
cd FirstProgram
```

Посмотрите файлы. Главные из них:

- `FirstProgram.csproj` — настройки проекта: тип SDK, target framework и свойства компиляции;
- `Program.cs` — исходный код;
- `obj` — промежуточные результаты сборки, создаются автоматически;
- `bin` — готовые результаты сборки, создаются автоматически.

Каталоги `bin` и `obj` не редактируют вручную и не коммитят: их можно восстановить сборкой.

## 5. Первая строка C#

В современном шаблоне `Program.cs` содержит примерно следующее:

```csharp
Console.WriteLine("Hello, World!");
```

Это полная минимальная программа благодаря механизму top-level statements. Разбор:

- `Console` — стандартный тип для консольного ввода и вывода;
- точка `.` открывает доступ к члену типа;
- `WriteLine` — метод, то есть именованное действие;
- круглые скобки содержат передаваемые действию данные;
- текст в двойных кавычках — строковый литерал;
- `;` завершает инструкцию.

Запустите:

```powershell
dotnet run
```

Ожидаемый вывод:

```text
Hello, World!
```

Команда сначала при необходимости собирает проект, затем запускает его. Отдельная проверка сборки:

```powershell
dotnet build
```

Успех обозначается `Build succeeded`. Слово warning означает предупреждение: сборка возможна, но компилятор заметил потенциальную проблему. Error означает, что сборка не завершилась.

## 6. Данные, переменная и ввод пользователя

Замените код:

```csharp
Console.Write("Введите ваше имя: ");
string? name = Console.ReadLine();
Console.WriteLine($"Здравствуйте, {name}!");
```

Что происходит по порядку:

1. `Write` печатает приглашение без перехода на новую строку.
2. `ReadLine` ждёт ввод и возвращает текст либо отсутствие текста при закрытом вводе.
3. `string?` означает «текст или отсутствие значения».
4. `name` — имя переменной. Переменная связывает имя со значением определённого типа.
5. `=` сохраняет результат правой части в переменной слева.
6. `$"...{name}..."` — интерполированная строка: значение вставляется в текст.

Запустите несколько раз с разными именами. Затем закройте кавычку или удалите `;` и выполните `dotnet build`. Это намеренная ошибка: найдите код, файл и номер строки в диагностике. Верните код в рабочее состояние.

### Важное правило чтения ошибки

Первая ошибка часто полезнее следующих: одна пропущенная скобка может запутать разбор всего оставшегося файла. Исправьте первую понятную причину и соберите снова.

## 7. Проект и solution

**Проект** отвечает на вопрос: «Как собрать эту программу или библиотеку?» Один проект создаёт один основной результат.

**Solution** объединяет проекты. Например, приложение и проект автоматических тестов могут находиться в одном solution.

Вернитесь в каталог урока и выполните:

```powershell
dotnet new sln -n Lesson01
dotnet sln Lesson01.sln add FirstProgram/FirstProgram.csproj
dotnet build Lesson01.sln
```

Первая команда создаёт solution, вторая регистрирует проект, третья собирает всё решение. Solution не копирует исходники и не превращает проекты в один — он хранит список и конфигурацию совместной работы.

## 8. Зачем нужен Git

Обычное сохранение файла помнит только текущую версию. **Git** хранит историю осмысленных изменений.

- **Repository** — рабочие файлы и скрытая история `.git`.
- **Working tree** — текущие файлы, которые вы видите.
- **Staging area** — список изменений, выбранных для следующего снимка.
- **Commit** — именованный снимок выбранных изменений.
- **Branch** — линия коммитов для отдельной работы.

Полезный цикл:

```powershell
git status
git add FirstProgram Lesson01.sln
git commit -m "lesson-01: add first console application"
```

`status` безопасен и показывает состояние. `add` не отправляет файлы в интернет — только выбирает их для коммита. `commit` сохраняет локальный снимок. Для отправки в удалённый репозиторий нужна отдельная команда `push`.

Перед `add` убедитесь, что `bin` и `obj` игнорируются `.gitignore`, а секретов в файлах нет.

## 9. Учебная задача как маленькая итерация

Большая фраза «выучить C#» не помогает решить, что делать сегодня. Её превращают в маленький проверяемый результат.

Пример цели: «Пользователь запускает консольную программу, вводит имя и видит персональное приветствие».

**Критерии приёмки** описывают конкретный результат:

- приложение запускается через `dotnet run`;
- программа просит имя;
- введённое имя появляется в приветствии;
- инструкция запуска записана в README.

**Definition of Done (DoD)** — общие правила качества для всех работ группы:

- solution собирается;
- тесты проходят, если они есть;
- нет секретов и build artifacts;
- код и запуск понятны проверяющему;
- изменения сохранены осмысленным коммитом.

Критерии приёмки отвечают «что должна уметь эта задача», DoD — «какому общему уровню качества соответствует любая завершённая работа».

## 10. Scrum без ритуалов ради ритуалов

Scrum — фреймворк коротких циклов для работы в условиях неопределённости. Он основан на трёх идеях:

- **прозрачность** — состояние работы видно;
- **проверка** — результат регулярно смотрят и сравнивают с целью;
- **адаптация** — план меняют по полученной информации.

В учебном курсе одна тема похожа на короткий Sprint:

1. понять цель и критерии;
2. выбрать небольшой план;
3. создать работающий increment — проверяемое улучшение;
4. получить review преподавателя;
5. провести retrospective: как улучшить способ работы.

Названия ролей и встреч менее важны, чем короткая обратная связь. Scrum не означает «делать без плана» и не требует ежедневно доказывать занятость.

## 11. Как проходит ревью

Ревью проверяет артефакт, а не ценность человека. Преподаватель может отметить:

- **обязательное замечание** — нарушено требование или есть дефект;
- **рекомендацию** — работа корректна, но есть более ясный вариант;
- **вопрос** — нужно понять ход решения.

На замечание отвечают новым изменением либо аргументом. Фраза «я выбрал так, потому что...» полезнее молчаливого согласия. При этом аргумент должен опираться на поведение и требования.

## 12. Итоговая карта

```text
Задача человека
  ↓ уточняем критерии
Исходный код C# в проекте
  ↓ dotnet build / компилятор
Собранное приложение .NET
  ↓ dotnet run / runtime
Наблюдаемый результат
  ↓ Git commit
Версия для проверки и обратной связи
```

Вы готовы к следующему уроку, если можете самостоятельно создать console project, изменить `Program.cs`, запустить его, прочитать простую ошибку и объяснить разницу между C#, .NET, проектом и Git-коммитом.
