# Урок 06. LINQ: запросы, выполнение и производительность

> LINQ — единая модель преобразования последовательностей. Главное — понимать не названия операторов, а форму потока данных, момент выполнения и цену повторного перечисления.

## Результаты обучения

- строить читаемые query pipelines
- различать deferred и immediate execution
- выбирать операторы по кардинальности результата
- находить повторное перечисление и лишнюю материализацию

## Карта темы

### 1. Модель IEnumerable

LINQ to Objects строится вокруг `IEnumerable<T>` и итератора. Большинство операторов возвращает описание вычисления, которое запускается при перечислении.

### 2. Фильтрация и проекция

`Where` сохраняет подходящие элементы, `Select` меняет форму, `SelectMany` разворачивает вложенные последовательности. Порядок операторов влияет и на смысл, и на объём работы.

### 3. Кардинальность

`First`, `Single`, `Last` и варианты `OrDefault` выражают разные ожидания. `Single` проверяет уникальность; `First` лишь выбирает первый элемент.

### 4. Материализация

`ToList`, `ToArray`, `ToDictionary` выполняют запрос и создают снимок. Материализация нужна для стабильности или повторного доступа, но преждевременная тратит память.

### 5. Агрегация

`Count`, `Sum`, `Min`, `Max`, `Average`, `Aggregate` сворачивают поток. Пустые последовательности и nullable-значения имеют разные правила, которые нужно проверять.

### 6. Группировка и соединения

`GroupBy` создаёт группы, `Join` сопоставляет ключи, `GroupJoin` моделирует one-to-many/left join вместе с `DefaultIfEmpty`. Ключи используют equality comparer.

### 7. Query syntax

Синтаксис `from` компилируется в вызовы операторов и удобен для нескольких join/let. Method syntax обычно компактнее для линейной цепочки; их можно смешивать.

### 8. IQueryable

`IQueryable<T>` хранит expression tree, который провайдер переводит, например, в SQL. Не вся C#-логика переводима; семантика и стоимость отличаются от LINQ to Objects.

### 9. Производительность

Повторное перечисление заново выполняет запрос и может повторить I/O. Следует фильтровать рано, не сортировать без нужды и избегать квадратичных вложенных поисков.

## Минимальный пример

```csharp
var report = orders
    .Where(o => o.CreatedAt >= periodStart)
    .SelectMany(o => o.Lines)
    .GroupBy(line => line.ProductId)
    .Select(g => new { ProductId = g.Key, Revenue = g.Sum(x => x.Price * x.Quantity) })
    .OrderByDescending(x => x.Revenue)
    .ToList();
```

Разберите пример построчно: какие гарантии выражены типами, где проверяются предусловия и что увидит вызывающий код. Затем измените один инвариант и проследите, какая часть публичного контракта должна поменяться.

## Типичные ошибки

- перечислять один внешний запрос много раз;
- использовать `SingleOrDefault` там, где дубликаты допустимы;
- вызывать `ToList` после каждого оператора;
- ожидать одинакового поведения IQueryable и IEnumerable;

## Итог

Тема считается усвоенной, когда вы можете не только применить механизм, но и объяснить его цену, область корректности и более простой альтернативный вариант. Проверяйте решения сборкой, тестами и небольшими экспериментами, а не только чтением.
