# Практика — урок 06

## Цель

Построить отчёты поверх каталога урока 05 и для каждого запроса объяснить форму данных, кардинальность и момент выполнения.

## Подготовка

Создайте не менее 12 книг:

- минимум 4 автора;
- повторяющиеся и уникальные теги;
- несколько ценовых диапазонов;
- один автор с несколькими книгами;
- издатели, один из которых не имеет книг.

Данные создаются в памяти. Доступ к БД пока не нужен.

## Обязательные запросы

1. Названия книг дешевле заданной суммы: `Where` + `Select`.
2. Все уникальные теги без учёта регистра: `SelectMany` + `Distinct`.
3. Книги по автору в порядке цены, затем названия: `OrderBy` + `ThenBy`.
4. Ровно одна книга по уникальному inventory number: обоснованный `SingleOrDefault`.
5. Первая книга, подходящая для рекомендации: `FirstOrDefault`.
6. Наличие хотя бы одной дорогой книги и проверка, что все цены неотрицательны: `Any` и `All`.
7. Общая и средняя цена с явным поведением для пустого набора.
8. Отчёт по авторам: имя, число книг, минимальная, максимальная и общая цена.
9. Сопоставление книг и издателей через `Join`.
10. Издатели вместе с числом книг, включая издателя без книг: `GroupJoin` либо осмысленный альтернативный индекс.

## Deferred execution experiment

1. Создайте запрос к изменяемому списку, но не перечисляйте.
2. Добавьте подходящую книгу.
3. Перечислите и сохраните результат.
4. Создайте `snapshot = query.ToList()`.
5. Добавьте ещё одну книгу.
6. Сравните повторный query и snapshot.

В README объясните результат через момент перечисления. Не ограничивайтесь фразой «LINQ ленивый».

## Собственный iterator

Напишите:

```csharp
IEnumerable<Book> TakeUntilPrice(
    IEnumerable<Book> source,
    decimal maximumPrice)
```

Метод выдаёт элементы через `yield return` и останавливается (`yield break`) при первой книге дороже границы. Опишите, почему это не то же самое, что `Where`.

## Ограничения

- не вызывать `ToList` после каждого оператора;
- не перечислять один потенциально дорогой запрос без причины несколько раз;
- не использовать side effects внутри `Where` и `Select`;
- не выбирать `First`/`Single` случайно;
- обработать пустые данные;
- сложная цепочка разбивается именованными переменными или методами, если перестаёт читаться.

## Документация запросов

Для каждого из десяти запросов заполните:

| Поле | Значение |
|---|---|
| Тип источника | Например, `IEnumerable<Book>` |
| Тип результата | Например, `List<string>` |
| Ожидаемая кардинальность | ноль/один/много |
| Deferred или immediate | и где выполняется |
| Поведение на пустом наборе | результат или ошибка |

## Критерии приёмки

- реализованы все десять запросов;
- результаты подтверждены ожидаемыми примерами;
- deferred experiment объяснён корректно;
- iterator действительно ленив и останавливается на границе;
- сортировка использует ThenBy для второго ключа;
- пустой набор не вызывает случайной необработанной ошибки;
- ученик может заменить любой запрос обычным циклом и объяснить его цену.

## Дополнительное задание

Реализуйте собственные `WhereSimple` или `DistinctBySimple` через iterator method. Сравните поведение с библиотечным оператором на пустом источнике, повторном перечислении и исключении predicate. Цель — понять протокол, а не заменить стандартную библиотеку в рабочем коде.
