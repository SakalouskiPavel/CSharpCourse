# Глоссарий начинающего C#/.NET-разработчика

Определения здесь намеренно простые. Точная профессиональная модель постепенно дополняется в уроках.

## А–Г

**Абстракция (abstraction)** — упрощённое представление, показывающее важное для текущей задачи и скрывающее лишние детали. Кнопка «Печать» является абстракцией сложной работы принтера.

**Алгоритм (algorithm)** — конечная последовательность понятных действий, преобразующая входные данные в результат.

**Аргумент (argument)** — конкретное значение, переданное методу при вызове. В `Print("Привет")` текст — аргумент.

**Библиотека (library)** — готовый код, который другая программа вызывает для решения типовых задач.

**Ветка Git (branch)** — отдельная линия изменений репозитория. Позволяет выполнять задание, не меняя основную версию до проверки.

**Выражение (expression)** — часть кода, которая вычисляет значение: `2 + 3`, `name.Length`, `isReady && hasData`.

**Git** — система контроля версий: сохраняет историю файлов, сравнивает изменения и помогает нескольким людям работать вместе.

## Д–К

**Данные (data)** — информация, с которой работает программа: числа, текст, даты, изображения, заказы.

**Зависимость (dependency)** — объект, библиотека или система, нужная другому коду для работы.

**Инвариант (invariant)** — правило, которое обязано оставаться истинным для корректного объекта. Например, баланс счёта не может стать отрицательным, если овердрафт запрещён.

**Интерфейс пользователя (UI, user interface)** — часть программы, через которую человек взаимодействует с ней: окно, сайт или командная строка.

**Исключение (exception)** — объект, сообщающий о невозможности нормально продолжить выполняемую операцию.

**Исходный код (source code)** — текст программы, написанный человеком.

**Класс (class)** — описание вида объектов: какие данные они хранят и какие действия выполняют.

**Клиент (client)** — код или программа, использующая услугу другого компонента или сервера.

**Коллекция (collection)** — объект для хранения группы значений.

**Коммит (commit)** — сохранённый в Git именованный снимок набора изменений вместе с автором и временем.

**Компилятор (compiler)** — инструмент, проверяющий код по правилам языка и преобразующий его для дальнейшего выполнения.

**Консоль (console)** — текстовый интерфейс ввода и вывода программы.

**Контракт (contract)** — обещание кода внешнему пользователю: что можно передать, что вернётся и какие условия соблюдаются.

## М–П

**Метод (method)** — именованный блок кода, выполняющий действие и, возможно, возвращающий результат.

**Объект (object)** — конкретный экземпляр типа во время работы программы. Класс `Book` — описание, конкретная книга — объект.

**Операционная система (OS)** — основная программа компьютера, управляющая файлами, памятью, устройствами и запуском процессов.

**Параметр (parameter)** — именованное место во время объявления метода, куда при вызове попадёт аргумент.

**Переменная (variable)** — именованное место, с которым программа связывает значение определённого типа.

**Платформа .NET** — runtime, библиотеки и инструменты для создания и выполнения приложений .NET.

**Поток (thread)** — последовательность выполняемых инструкций внутри процесса. Несколько потоков одного процесса разделяют его память.

**Проект (project)** — исходники и настройки одной собираемой программы или библиотеки; в .NET настройки обычно находятся в `.csproj`.

**Процесс (process)** — запущенный экземпляр программы с выделенными операционной системой ресурсами.

## Р–Я

**Репозиторий (repository)** — каталог проекта вместе с историей Git. В архитектуре этим словом также называют объект доступа к данным; смысл определяется контекстом.

**Runtime / среда выполнения** — часть платформы, которая загружает и исполняет скомпилированную программу, управляет памятью и исключениями.

**Сборка (build)** — процесс проверки исходников и получения запускаемых файлов или библиотеки. Словом assembly также называют один из результатов сборки .NET.

**Сериализация (serialization)** — преобразование данных объекта в формат для передачи или хранения, например JSON.

**Сервер (server)** — программа, принимающая запросы клиентов и предоставляющая данные или операции.

**Синтаксис (syntax)** — правила записи конструкций языка: где ставятся скобки, точки с запятой и ключевые слова.

**Solution** — файл и логическая группа связанных .NET-проектов.

**Тип (type)** — описание допустимого вида значения и операций над ним: целое число, строка, дата, заказ.

**Тест (test)** — код или процедура, проверяющие ожидаемое поведение на выбранном сценарии.

**Фреймворк (framework)** — набор библиотек и правил, который задаёт каркас приложения и вызывает пользовательский код в предусмотренных точках.

**API (Application Programming Interface)** — программный интерфейс: доступные другим компонентам операции, форматы и правила взаимодействия.

**CLI (Command-Line Interface)** — текстовый интерфейс командной строки. `dotnet` и `git` являются CLI-инструментами.

**IDE (Integrated Development Environment)** — интегрированная среда разработки: редактор кода, подсказки, запуск, отладка и навигация.

**SDK (Software Development Kit)** — комплект инструментов разработчика. .NET SDK включает компилятор, CLI и шаблоны.
