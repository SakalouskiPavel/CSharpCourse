# Критерии проверки практики

Каждая категория оценивается от 0 до 2 баллов. Базовая работа считается принятой при результате не ниже 8 из 12 и отсутствии критических дефектов.

| Категория | 0 | 1 | 2 |
|---|---|---|---|
| Корректность | Не работает | Частично работает | Все сценарии и границы учтены |
| Владение темой | Решение случайно/скопировано | Основная идея понятна | Решения уверенно объяснены |
| Дизайн | Сильная связанность | Есть спорные места | Ответственности и зависимости ясны |
| Качество кода | Трудно читать | В целом читаемо | Имена, стиль и простота на хорошем уровне |
| Тесты | Нет полезных тестов | Покрыт happy path | Покрыты важные ветви и ошибки |
| Git и документация | Сдача непрозрачна | Запустить можно | История и инструкция помогают ревью |

## Критические дефекты

- проект не собирается или не запускается по инструкции;
- обязательная часть задания отсутствует;
- секреты или персональные данные попали в репозиторий;
- ученик не может объяснить ключевую реализацию;
- тесты намеренно отключены либо результат подменён.

## Формат обратной связи

Замечание должно содержать место, наблюдаемую проблему, её последствие и ожидаемое направление исправления. Преподаватель разделяет обязательные замечания, рекомендации и вопросы для обсуждения. Дополнительное задание назначается, когда пробел нельзя надёжно проверить правкой текущего решения.
