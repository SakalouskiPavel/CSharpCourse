# Как проходить курс

## Цикл работы над уроком

1. Обновите свою учебную ветку и прочитайте `ТЕОРИЯ.md`.
2. Письменно или устно ответьте на вопросы из `ВОПРОСЫ.md`. Не подглядывайте в конспект до первой попытки.
3. В каталоге урока создайте solution: `dotnet new sln`.
4. Создайте проекты, которые требует `ПРАКТИКА.md`, и добавьте их в solution.
5. Делайте небольшие осмысленные коммиты. Перед сдачей выполните сборку и тесты.
6. Заполните описание сдачи: что сделано, как запустить, какие решения приняты, что осталось спорным.
7. Отправьте ветку преподавателю или создайте pull request — согласно правилам учебной группы.

## Имена веток и коммитов

Рекомендуемое имя ветки: `student/<имя>/lesson-XX`. Примеры сообщений:

- `lesson-06: add query pipeline`;
- `lesson-06: cover grouping with tests`;
- `lesson-06: address review comments`.

Один финальный коммит допустим для маленькой работы, но история из нескольких содержательных шагов лучше показывает ход мысли. Не коммитьте `bin`, `obj`, секреты, дампы БД и пользовательские настройки IDE.

## Минимальная планка сдачи

- solution собирается командой `dotnet build` без ошибок;
- автоматические тесты проходят через `dotnet test`;
- нет предупреждений, которые ученик не может объяснить;
- входные данные проверяются, ошибки обрабатываются предсказуемо;
- публичные имена передают смысл, форматирование единообразно;
- README проекта объясняет запуск, если он не очевиден;
- выполнены обязательные требования из практики.

## Работа после ревью

Замечания преподавателя исправляются новыми коммитами — историю не нужно переписывать, если преподаватель не попросил об этом. На каждое замечание следует либо изменение, либо аргументированный ответ. После повторной проверки урок получает статус: «принято», «принято с рекомендациями» или «на доработку».

## Академическая честность и ИИ

Документацию, поисковики и ИИ использовать можно, если правила группы не говорят обратного. Ученик обязан понимать и уметь объяснить каждую строку сдаваемого решения, назвать альтернативы и воспроизвести ключевую часть без подсказки. Источник нетривиального заимствования указывается в README проекта.
