# Урок 13. Делегаты и события

> Делегат — типизированная ссылка на вызываемое поведение; событие — ограниченный механизм публикации уведомлений. Они лежат в основе callbacks, LINQ, middleware и асинхронных API.

## Результаты обучения

- объявлять и комбинировать делегаты
- понимать closure и variance
- проектировать события с контролем владельца
- избегать утечек подписок

## Карта темы

### 1. Тип делегата

Delegate описывает сигнатуру вызова. Method group преобразуется в подходящий delegate; `Action`, `Func` и `Predicate` покрывают общие случаи, именованный delegate добавляет предметный смысл.

### 2. Multicast

Делегаты immutable и могут содержать invocation list. Вызовы идут по порядку; возвращается результат последнего обработчика, а первое необработанное исключение прерывает цепочку.

### 3. Лямбды и closures

Лямбда может захватить переменные, которые помещаются в объект closure. Захватывается переменная, не моментальное значение; это влияет на циклы, lifetime и аллокации.

### 4. Variance

Delegate с возвращаемым значением поддерживает covariance, с параметром — contravariance в безопасных позициях. Совместимость определяется типами, а не только одинаковой формой кода.

### 5. События

`event` позволяет внешнему коду только подписываться/отписываться; вызывать событие может владелец. Стандартный паттерн — `EventHandler<TEventArgs>` и защищённый метод On... при наследовании.

### 6. Подписка и lifetime

Publisher удерживает ссылку на subscriber через delegate. Долгоживущий publisher может вызвать memory leak; нужна отписка, IDisposable subscription или weak pattern в специальных случаях.

### 7. Потокобезопасный вызов

Локальная копия `handler = Event` либо null-conditional защищает от изменения между проверкой и вызовом. Это не делает состояние обработчиков потокобезопасным.

### 8. События и сообщения

In-process event синхронен и связан временем жизни. Для долговечных/распределённых сообщений нужны другие гарантии, durable broker и явная обработка ошибок.

## Минимальный пример

```csharp
public sealed class TemperatureSensor
{
    public event EventHandler<TemperatureChangedEventArgs>? TemperatureChanged;

    public void Update(decimal value) =>
        TemperatureChanged?.Invoke(this, new(value));
}
```

Пример — отправная точка, а не готовая архитектура. Проверьте предусловия, lifetime объектов, обработку ошибок и поведение на границах. Для инфраструктурных тем отдельно фиксируйте, какие гарантии даёт библиотека, а какие должен обеспечить ваш код.

## Типичные ошибки

- публично выставлять delegate-поле вместо event;
- забывать отписку от долгоживущего publisher;
- полагаться на return value multicast delegate;
- захватывать изменяемую loop variable без понимания lifetime;
