# Урок 13. Делегаты, лямбды и события

> Обычный вызов жёстко называет метод. Делегат позволяет передать действие как значение: сохранить его в переменной, получить параметром и вызвать позже. Событие ограничивает этот механизм для уведомления подписчиков.

## 1. Метод как значение

```csharp
static int Double(int number) => number * 2;

Func<int, int> operation = Double;
int result = operation(5); // 10
```

`Func<int,int>` — delegate type: принимает `int`, возвращает `int`. В `operation` хранится ссылка на совместимый метод вместе с target object, если метод экземплярный.

Переменная не содержит результат `Double(5)`, она содержит вызываемое поведение. Скобки `operation(5)` выполняют вызов.

## 2. Объявление собственного delegate

```csharp
public delegate decimal DiscountRule(decimal total);

static decimal NoDiscount(decimal total) => total;
static decimal TenPercent(decimal total) => total * 0.9m;

DiscountRule rule = TenPercent;
decimal finalTotal = rule(100m);
```

Delegate declaration задаёт сигнатуру: тип результата и параметры. Именованный delegate полезен, когда сигнатура имеет предметный смысл или нужны специальные annotations.

Два delegate types с одинаковой сигнатурой остаются разными типами, хотя method group может преобразоваться в каждый совместимый.

## 3. `Action`, `Func`, `Predicate`

- `Action`/`Action<T...>` — действие без результата (`void`);
- `Func<T..., TResult>` — последний generic parameter является результатом;
- `Predicate<T>` — исторически распространённая форма `T → bool`.

```csharp
Action<string> print = Console.WriteLine;
Func<decimal, decimal, decimal> add = (left, right) => left + right;
Predicate<Book> isExpensive = book => book.Price >= 50m;
```

Для локальной технической функции стандартные types уменьшают число объявлений. Для публичного предметного контракта интерфейс или именованный delegate может быть яснее.

## 4. Method group, anonymous method и lambda

```csharp
Func<int, int> first = Double;
Func<int, int> second = delegate (int number) { return number * 2; };
Func<int, int> third = number => number * 2;
```

Три формы создают совместимое вызываемое значение. Lambda может иметь block body:

```csharp
Func<int, int> safeDouble = number =>
{
    if (number > int.MaxValue / 2)
        throw new ArgumentOutOfRangeException(nameof(number));

    return number * 2;
};
```

Тип параметров часто выводится из target delegate. Lambda без target type в старом/неоднозначном контексте может потребовать явного типа.

## 5. Зачем передавать поведение

```csharp
static List<Book> Filter(
    IEnumerable<Book> source,
    Func<Book, bool> condition)
{
    var result = new List<Book>();

    foreach (Book book in source)
        if (condition(book))
            result.Add(book);

    return result;
}
```

```csharp
List<Book> cheap = Filter(books, book => book.Price < 20m);
```

Алгоритм обхода отделён от изменяемого правила. Именно такая идея лежит под `Where`, `Select`, sort comparison, callbacks и middleware.

## 6. Delegate instance: метод и target

```csharp
public sealed class PricePolicy
{
    public decimal AddTax(decimal price) => price * 1.2m;
}

var policy = new PricePolicy();
Func<decimal, decimal> calculator = policy.AddTax;
```

Delegate удерживает ссылку на `policy`, чтобы позже вызвать instance method. Пока жив delegate, target object обычно достижим для GC. Это важно для событий и утечек подписок.

Static method target object не требует.

## 7. Multicast delegate

Delegate immutable. Оператор `+=` создаёт новый delegate с invocation list:

```csharp
Action<string> handlers = WriteToConsole;
handlers += WriteToFile;
handlers("Запись");
```

Handlers вызываются по порядку списка. `-=` удаляет последнее совпадающее вхождение.

Если handler выбрасывает необработанное исключение, вызов прерывается и следующие не выполняются. Для multicast с return value вызывающий получает только результат последнего handler-а; поэтому такие списки обычно используют с `void`.

Если требуется изолировать ошибки каждого subscriber, publisher должен явно получить `GetInvocationList`, вызвать элементы отдельно и определить политику. Молчаливое проглатывание ошибок не является безопасностью.

## 8. Closure и захват переменной

```csharp
int multiplier = 2;
Func<int, int> multiply = number => number * multiplier;

multiplier = 3;
Console.WriteLine(multiply(10)); // 30
```

Lambda захватила переменную, а не снимок значения. Compiler создаёт hidden closure object, чтобы переменная жила после выхода из исходного scope.

Closure может:

- продлить lifetime объектов;
- создать allocation;
- разделить mutable state между вызовами;
- привести к race condition в конкурентном коде;
- дать неожиданный результат в цикле, если захвачена изменяемая переменная.

```csharp
var actions = new List<Action>();
for (int index = 0; index < 3; index++)
{
    int capturedIndex = index;
    actions.Add(() => Console.WriteLine(capturedIndex));
}
```

Локальная копия явно фиксирует значение каждой итерации.

## 9. Variance delegate types

Covariance результата: функция, возвращающая `Cat`, подходит там, где ожидается `Animal`.

Contravariance параметра: действие, умеющее обработать любой `Animal`, подходит там, где будут передавать только `Cat`.

```csharp
Func<Cat> createCat = () => new Cat();
Func<Animal> createAnimal = createCat;

Action<Animal> handleAnimal = animal => Console.WriteLine(animal.Name);
Action<Cat> handleCat = handleAnimal;
```

Безопасность та же, что у `out T`/`in T` интерфейсов урока 04.

## 10. Что добавляет `event`

Публичное delegate property/field позволило бы внешнему коду заменить всех подписчиков или вызвать их самостоятельно:

```csharp
// Плохая граница:
public Action? Changed;
```

`event` ограничивает внешнего клиента операциями подписки и отписки:

```csharp
public sealed class TemperatureSensor
{
    public event EventHandler<TemperatureChangedEventArgs>? TemperatureChanged;

    public void Update(decimal value)
    {
        TemperatureChanged?.Invoke(
            this,
            new TemperatureChangedEventArgs(value));
    }
}
```

Только containing type может вызвать event обычным способом. Внешний код:

```csharp
sensor.TemperatureChanged += OnTemperatureChanged;
sensor.TemperatureChanged -= OnTemperatureChanged;
```

## 11. Стандартный event pattern

```csharp
public sealed class TemperatureChangedEventArgs : EventArgs
{
    public decimal Value { get; }

    public TemperatureChangedEventArgs(decimal value)
    {
        Value = value;
    }
}
```

`EventHandler<TEventArgs>` имеет `(object? sender, TEventArgs args)`. `sender` указывает publisher, args несёт данные факта.

Событие называют свершившимся фактом: `OrderPaid`, `TemperatureChanged`. Если subscriber должен разрешить действие или вернуть обязательный результат, обычный метод/strategy часто честнее.

Для наследуемого publisher распространён protected virtual `OnTemperatureChanged`, но sealed class может вызывать event напрямую и проще контролирует контракт.

## 12. Время вызова и ошибки

Обычное .NET event синхронно: `Invoke` выполняет subscribers в текущем call stack. Медленный subscriber задерживает publisher; исключение может сорвать операцию.

Publisher должен определить:

- событие до или после изменения состояния;
- может ли subscriber отменить действие;
- порядок значим или нет;
- что происходит при исключении;
- допустима ли reentrancy (subscriber снова вызывает publisher).

Не называйте in-process event гарантированной системой сообщений: нет durability, retries, persistence и доставки после падения процесса.

## 13. Подписка и memory leak

Publisher хранит delegates, delegates удерживают subscriber targets:

```text
долгоживущий publisher → delegate → subscriber
```

Если subscriber должен исчезнуть раньше publisher, но не отписался, GC считает его достижимым. Это managed memory leak.

Решения:

- симметричная отписка;
- subscription object с `IDisposable`;
- общий ограниченный lifetime;
- weak event pattern в специальных UI/framework scenarios.

Anonymous lambda трудно отписать, если ссылка на тот же delegate instance не сохранена:

```csharp
EventHandler<TemperatureChangedEventArgs> handler = (_, args) => Print(args.Value);
sensor.TemperatureChanged += handler;
sensor.TemperatureChanged -= handler;
```

## 14. Потокобезопасность

```csharp
TemperatureChanged?.Invoke(this, args);
```

Compiler/оператор получает текущий delegate snapshot для вызова; одновременная подписка может повлиять на будущий, а не текущий список. Но это не делает state publisher-а или subscribers потокобезопасным.

Не удерживайте lock при вызове чужих handlers: subscriber может долго работать, повторно войти или попытаться взять другой lock. Обычно состояние изменяют под защитой, затем публикуют snapshot вне lock.

## 15. Async handlers

`event EventHandler` возвращает void и не умеет корректно ждать async lambda; получится `async void` с трудной обработкой ошибок.

Если publisher обязан дождаться обработчиков, лучше явный async contract:

```csharp
public delegate Task AsyncEventHandler<TEventArgs>(
    object sender,
    TEventArgs args,
    CancellationToken cancellationToken);
```

Затем publisher определяет последовательный/параллельный вызов, ошибки и cancellation. Называть это event или pipeline — вопрос API, но Task должен наблюдаться.

## 16. Delegate или interface

Delegate удобен для одной операции без сложной identity: predicate, transform, callback.

Interface лучше, если:

- операций несколько;
- важна предметная роль;
- реализация имеет lifecycle/state;
- контракт должен развиваться;
- нужны разные методы ошибок/возможностей.

`Func` в каждом constructor-е может сделать назначение зависимостей непонятным; именованный interface улучшает навигацию.

## Частые ошибки

- delegate field публикуется вместо event;
- multicast delegate с результатом используется как aggregation;
- исключение одного handler случайно останавливает остальных;
- closure захватывает mutable variable;
- long-lived publisher удерживает subscriber;
- lambda невозможно отписать из-за потерянного delegate instance;
- async lambda подписывается на void event без политики ошибок;
- event используется там, где нужен обязательный request/response;
- publisher вызывает handlers под lock;
- in-process event принимается за durable broker.

## Итог

```text
Delegate = типизированное вызываемое значение
Lambda   = краткая запись поведения
Closure  = захваченное окружение
Event    = delegate с внешним += / -= и владельцем вызова
```

Вы готовы дальше, если можете передать Func как стратегию, объяснить closure, подписаться/отписаться от event и определить политику ошибок/lifetime.
