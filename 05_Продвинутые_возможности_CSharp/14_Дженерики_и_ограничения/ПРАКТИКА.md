# Практика — Урок 14. Дженерики, реализация и ограничения

## Задача

Реализовать типобезопасный pipeline обработки сообщений с generic handlers, constraints и композицией результатов.

## Обязательные требования

- создать generic interface обработчика и dispatcher;
- исключить неверные комбинации типов на этапе компиляции;
- использовать минимум два осмысленных constraint;
- не применять reflection/dynamic для обычной маршрутизации;
- покрыть несколько типов сообщений тестами;

## Порядок работы

1. Запишите входы, выходы, ошибки и ограничения.
2. Создайте отдельные production- и test-проекты в solution.
3. Реализуйте минимальный сквозной сценарий и зафиксируйте коммитом.
4. Добавьте граничные случаи, диагностику и документацию запуска.
5. Выполните `dotnet build` и `dotnet test`.

## Критерии готовности

- обязательные сценарии воспроизводятся по README;
- выбранные конструкции усиливают контракт, а не маскируют дизайн;
- тесты проверяют успех, границы и ожидаемые ошибки;
- ресурсы, данные и исключения обрабатываются предсказуемо;
- ученик может объяснить альтернативу каждому нетривиальному решению.
