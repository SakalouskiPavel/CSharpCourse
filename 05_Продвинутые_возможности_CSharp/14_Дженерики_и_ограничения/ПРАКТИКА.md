# Практика — урок 14

## Цель

Создать типобезопасный pipeline команд и результатов без `object`, runtime casts и Reflection в обычной маршрутизации.

## Контракты

```csharp
public interface ICommand<TResult>
{
}

public interface ICommandHandler<TCommand, TResult>
    where TCommand : ICommand<TResult>
{
    TResult Handle(TCommand command);
}
```

Создайте минимум три пары command/result, например поиск книги, регистрация книги и расчёт статистики.

## Generic pipeline

Реализуйте decorators/composition functions для handler-а:

- validation перед выполнением;
- logging имени command/result type;
- timing без изменения результата.

Pipeline сохраняет точные TCommand/TResult. Запрещены `object`, `dynamic`, casts и Reflection для вызова.

## Generic Result

Создайте `Result<TValue,TError>` либо упрощённый `Result<T>` с явными success/failure factories. Он не позволяет прочитать Value при failure без определённого поведения.

Сравните с `default(T)` и объясните, почему Result отличает найденный `0` от отсутствия.

## Constraints

Добавьте осмысленные constraints минимум в двух местах. Для каждого в README ответьте:

- какую операцию он разрешает внутри generic-кода;
- какой неверный type argument запрещает;
- нельзя ли выразить требование проще.

Не добавляйте `new()` без реального parameterless construction.

## Variance experiment

Отдельно создайте `IProducer<out T>` и `IConsumer<in T>` с Animal/Cat/Dog. Покажите разрешённые присваивания и закомментированный compile error для небезопасного направления. Объясните List invariance через попытку добавить Dog.

## Проверки

- неверная command/handler pair не компилируется;
- result type выводится/сохраняется без cast;
- success/failure различаются для value/reference types;
- validation failure не вызывает inner handler;
- pipeline сохраняет ошибки/результаты;
- variance работает только в безопасном направлении.

## Критерии приёмки

- ordinary dispatch полностью type-safe;
- нет `if typeof(T)` веток;
- constraints связаны с используемыми capabilities;
- Result не полагается на default как marker;
- variance experiment объяснён;
- README сравнивает generic method, interface polymorphism и object;
- solution собирается.
