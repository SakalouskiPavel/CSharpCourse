# Урок 16. От препроцессора до указателей

> C# содержит низкоуровневые и условные механизмы, полезные в инфраструктуре и производительных участках. Их применение оправдано только измеримой задачей и ясной локализацией риска.

## Результаты обучения

- использовать директивы и conditional compilation
- понимать nullable context, checked и unsafe
- работать со Span, stackalloc и fixed
- оценивать цену низкоуровневой оптимизации

## Карта темы

### 1. Директивы препроцессора

`#if`, `#define`, `#error`, `#warning`, `#line`, `#region` обрабатываются до компиляции, но это не текстовый макропроцессор C. Символы задаются проектом и подходят для platform/build различий.

### 2. Nullable context

`#nullable enable` включает анализ nullable reference types. Аннотации — часть compile-time контракта и могут дополняться attributes вроде NotNullWhen.

### 3. checked/unchecked

Для integral arithmetic `checked` обнаруживает overflow, `unchecked` задаёт wraparound. Контекст и project settings важны; floating-point ведёт себя иначе.

### 4. Unsafe-код

`unsafe` разрешает pointers, address-of и dereference. CLR memory safety частично снимается; размер, lifetime, alignment и bounds становятся ответственностью разработчика.

### 5. fixed и pinning

GC перемещает объекты, поэтому адрес managed memory временно фиксируют. Долгое pinning фрагментирует heap; часто Span и MemoryMarshal дают безопаснее нужный контроль.

### 6. stackalloc

Выделяет небольшой буфер на стеке и обычно присваивается Span<T>. Размер входа должен быть ограничен, иначе возможен stack overflow.

### 7. Span и Memory

Span — stack-only view над contiguous memory и не может жить через await. Memory<T> можно хранить и передавать асинхронно; оба выражают срез без владения.

### 8. ref semantics

`ref return`, `ref local`, `ref readonly` избегают копирования или дают доступ к месту хранения. Они усложняют aliasing и требуют строгого lifetime analysis компилятора.

### 9. Interop

P/Invoke связывает native ABI: calling convention, layout, encoding и ownership должны совпасть. Source-generated interop уменьшает runtime marshalling.

## Минимальный пример

```csharp
Span<byte> buffer = stackalloc byte[16];
if (!Guid.TryParse(input, out var id))
    return false;
return id.TryWriteBytes(buffer);
```

Пример — отправная точка, а не готовая архитектура. Проверьте предусловия, lifetime объектов, обработку ошибок и поведение на границах. Для инфраструктурных тем отдельно фиксируйте, какие гарантии даёт библиотека, а какие должен обеспечить ваш код.

## Типичные ошибки

- скрывать большие различия логики каскадом #if;
- выделять stackalloc по неограниченному пользовательскому размеру;
- удерживать pin дольше необходимого;
- применять unsafe до профилирования;

## Итог

Умение использовать механизм включает понимание его runtime-модели, цены и альтернатив. Подтверждайте выводы небольшими тестами, логами, generated output или профилированием — в зависимости от темы.
