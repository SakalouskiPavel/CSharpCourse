# Урок 17. Полезные возможности новых версий C#

> Язык развивается короткими ежегодными итерациями. Цель урока — научиться оценивать новые конструкции по тому, какие ошибки они исключают и насколько ясно выражают намерение, а не собирать синтаксические трюки.

## Результаты обучения

- применять records, patterns и nullable references
- использовать required/init, primary constructors и collection expressions
- понимать привязку версии языка к SDK/TFM
- мигрировать код маленькими проверяемыми шагами

## Карта темы

### 1. Версия языка

SDK выбирает compiler, target framework — доступные API/runtime. `LangVersion` можно задать явно, но `latest` в воспроизводимой сборке способен неожиданно менять семантику.

### 2. Nullable reference types

Аннотации `?`, flow analysis и null-forgiving `!` делают допущения явными. `!` не выполняет runtime check и должен быть редким доказанным утверждением.

### 3. Records

Record class/struct дают value equality, print members, positional syntax и `with`. Они удобны для value-like данных, но shallow immutability не замораживает вложенные коллекции.

### 4. init и required

`init` ограничивает присваивание фазой инициализации, `required` требует указать член при создании. Конструкторные инварианты всё ещё нужны для связанных значений.

### 5. Pattern matching

Property, relational, logical, list и recursive patterns компактно описывают форму. Exhaustive switch expression помогает моделировать закрытые варианты, но guards должны оставаться читаемыми.

### 6. Primary constructors

Параметры доступны во всём типе и уменьшают boilerplate. Они не всегда становятся полями; публичный API и capture следует понимать явно.

### 7. File-scoped и global using

Уменьшают структурный шум. Global using должен отражать действительно общую зависимость, иначе происхождение типов становится неочевидным.

### 8. Collection expressions

`[1, 2, 3]` и spread `..` создают коллекцию в target-typed контексте. Конкретная стратегия зависит от целевого типа; читаемость важнее новизны.

### 9. Raw strings и UTF-8 literals

Raw string literals удобны для JSON/SQL без escaping, количество кавычек определяет delimiter. `"text"u8` создаёт UTF-8 byte span без runtime encoding.

### 10. Миграция

Включайте nullable отдельно, исправляйте предупреждения по границам, добавляйте analyzers и тесты. Новая конструкция полезна, если уменьшает состояние ошибки и понятна команде.

## Минимальный пример

```csharp
public sealed record OrderLine(string Sku, int Quantity)
{
    public decimal Total(decimal price) => Quantity switch
    {
        <= 0 => throw new InvalidOperationException("Quantity must be positive"),
        _ => price * Quantity
    };
}
```

Пример — отправная точка, а не готовая архитектура. Проверьте предусловия, lifetime объектов, обработку ошибок и поведение на границах. Для инфраструктурных тем отдельно фиксируйте, какие гарантии даёт библиотека, а какие должен обеспечить ваш код.

## Типичные ошибки

- ставить LangVersion latest без контроля SDK;
- подавлять nullable warnings оператором !;
- считать record глубоко immutable;
- переписывать стабильный код только ради нового синтаксиса;

## Итог
